# quantum-random-generator
A modular Quantum Random Number Generator (QRNG) powered by Qiskit and Quantum Hardware.

## Features
* ** Hardware-Ready :** Integrate with IBM Quantum backends.
* ** Modular :** Separated logic for backend management and circuit generation.
* ** Flexible :** Generate secure passwords or raw entropy keys.

## Quick start
1. Clone the repository. git clone https://github.com/iamrealkrrish/quantum-random-generator.git
2. Install dependencies: ' pip install -r requirements.txt '
3. Run the application: ' python main.py '

## Architecture
The project follows a modular structure to keep hardware-specific code separated from the core generator logic.

## Hardware Mode & Known Limitations

This project supports execution on real IBM Quantum hardware. However, please be aware of the following constraints:

* **Qubit Limitations:** Real hardware has a finite number of physical qubits (e.g., 5 to 127 qubits depending on the backend). 
* **Current Status:** The hardware execution path has been architected to automatically "chunk" requests to fit within the hardware's qubit limits.
* **Help Wanted:** While the logic is implemented to handle hardware constraints, I have not yet performed full-scale testing on live IBM Quantum hardware. 
    * If you have access to an IBM Quantum account, please try running the hardware mode. 
    * If you encounter issues (e.g., `CircuitTooWideForTarget` or job queue errors), please open an issue and include the **backend name** you selected and the **error message** you received.
* **Usage Tip:** For large keys (e.g., 128+ bits), the local `qasm_simulator` is recommended. Use the "Hardware Mode" only if you are testing connectivity or specifically require true quantum entropy.
